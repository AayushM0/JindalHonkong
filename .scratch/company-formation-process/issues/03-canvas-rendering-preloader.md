## Answer
We fully implemented the preload and canvas rendering engine:
- Preloading is lazy-initiated via an `IntersectionObserver` with a `200px` rootMargin triggering when approaching the wrapper viewport.
- Priority preloads frames 1-10 inside a `Promise.all` pool, displaying a skeleton loader until complete.
- Decodes all frames asynchronously using `Image.prototype.decode()` before caching them in a mutable `useRef` array, preventing layout thrashing or browser main-thread blocking.
- Implemented responsive canvas Cover scaling math (`object-fit: cover` equivalent) supporting multi-DPR scaling up to `2x`.
- Drawing frames is performed imperatively, directly reading from the decoded cache and writing to the canvas context, which bypasses React component updates entirely.
