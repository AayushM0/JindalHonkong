## Answer
We integrated GSAP ScrollTrigger for pinned scrollytelling:
- Registered GSAP's `ScrollTrigger` plugin.
- Used `gsap.matchMedia()` to set up different height/pin rules dynamically based on viewport width:
  - Desktop (>=768px): Wrapper set to `400vh` scroll duration, pinning the inner `.process-pin` container.
  - Mobile (<768px): Wrapper set to `240vh` scroll duration, pinning the container.
- Inside the ScrollTrigger `onUpdate` handler, mapped `progress` directly to the frame sequence index (1-80) and updated the canvas frame drawing.
- Handled the mobile sampling path by snapping indices to odd numbers so that the loader skips drawing missing (even) frames.
- Handled ScrollTrigger cleanup within the effect's return callback using `mm.revert()`, ensuring instances are destroyed on unmount.
